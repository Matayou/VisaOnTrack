# Context Snapshots

**Purpose:** Periodic snapshots of project context to prevent information loss  
**Created:** 2025-01-11

---

## üìã What Are Context Snapshots?

Context snapshots are periodic captures of the current project state, including:
- Current status summary
- Active tasks and assignments
- Blockers and resolutions
- Key decisions made
- Agent statuses
- Next actions

**Why:** Prevent context loss when:
- Chat context window gets full
- Starting new chat instances
- Resuming work after long breaks
- Multiple agents working on different tasks

---

## üìÖ Snapshot Schedule

### Weekly Snapshots
- **When:** Every Friday (end of week)
- **Purpose:** Capture weekly progress
- **File:** `CONTEXT_SNAPSHOT_YYYY-MM-DD.md`

### Event-Based Snapshots
- **When:** After major milestone completions
- **When:** After significant decisions
- **When:** Before starting new major work
- **When:** Before archiving chat (80% full)

---

## üìö How to Use

### When Starting New Chat
1. Read latest context snapshot
2. Read COORDINATION_HUB.md
3. Read AGENT_STATUS_BOARD.md
4. Acknowledge current status

### When Resuming Work
1. Check latest context snapshot
2. Check coordination hub for updates
3. Review recent git commits
4. Resume work with full context
